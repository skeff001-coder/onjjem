Contrado Helix API Usage Guide
Overview
The Contrado Print on demand (Helix) API is a RESTful HTTP API that uses JSON request and response bodies.

Supported HTTP methods:

GET
POST
PUT
DELETE
Making Requests
Request Format
All requests must include:

X-API-Key: <private_token>
Content-Type: application/json
Example Request
curl -X GET "https://{base_url}/helix/v1/{request_path}" \
  -H "Content-Type: application/json" \
  -H X-API-Key: <private_token>
Response Format
Responses are returned in JSON format with appropriate HTTP status codes.

Successful Response (200 OK)
{
  "success": true,
  "message":"Request Successful",
  "data": {//API Endpoint return data
	//....
  },
   "error": null,
    "timestamp": "2026-03-08T10:04:41.1512249Z",
    "path": "/api/v1/stores",
    "version": "1.0"
}
Error Response
{
  "success": false,
  "message":"error messge ex: Invalid argument provided",
  "data": null,
   "error": {// error object},
    "timestamp": "2026-03-08T10:04:41.1512249Z",
    "path": "/api/v1/stores",
    "version": "1.0"
}
Timestamps
All timestamps are returned in UTC using ISO 8601 format.

HTTP Status Codes
Status Code	Description
200 OK	Request successful
201 Created	Resource created successfully
400 Bad Request	Invalid request data
401 Unauthorized	Authentication required or invalid token
403 Forbidden	Insufficient scope or permissions
404 Not Found	Resource not found
429 Too Many Requests	Rate limit exceeded
500 Internal Server Error	Unexpected server error
Rate Limiting
The Helix API applies a general limit of 100 requests per minute.

Authentication
Private Tokens
Private tokens are used for API authentication.

Use a private token if you are building an integration for your own store and only your own services will call the API.

Client Types
There are two client types:

Store: can access only one store.
Account: can access all stores under the account. For store-specific endpoints, include X-Store-ID.
Scopes
Scope	Client Type	Description
StoresRead	Store, Account	Read stores
StoresCollectionRead	Store, Account	Read store collections
StoreProductsRead	Store, Account	Read store products
StoreOrdersRead	Store, Account	Read store orders
StoreOrdersWrite	Store, Account	Create and manage store orders
Using Private Tokens
Private tokens can be generated in Contrado Store Account → API Integration.

Notes:

Tokens do not require refresh.
Tokens remain valid until expiration or manual deletion.
Copy the token when created; it cannot be viewed again.
Store-Level Token Example
curl --location \
    --request GET 'https://{base_url}/helix/v1/{request_path}' \
    --header 'X-API-Key: {private_token}'
     --header 'X-Store-Id: {store_id}'
Account-Level Token Example (with Store Context)
curl --location \
    --request GET 'https://{base_url}/helix/v1/{request_path}' \
    --header 'X-API-Key: {private_token}' \  
Order Creation Validation Rules (Required)
For POST /orders/create, requests should be rejected when any of the following occurs:

Missing shipping address (if required by product/order type)
Invalid, deleted, or unsupported product
Product does not belong to the authorized store/account
Option/value mismatch for product configuration
Empty line items
Invalid quantity or pricing format
Invalid culture code

Field Descriptions
Field Name	Type	Description
productThumb	String	The primary, customer-facing thumbnail image (capped at 590px resolution). Intended Use: Product listing pages, grids, and catalogs.
productImages	Array of Objects	Contains the 2D image views/design templates of the product. Includes High Quality (thumbHighQuality ), 800px (storeThumb800Quality) and 420px (storeThumb420Quality) resolutions. Intended Use: Product Detail Pages (PDP).
product3dImages	Array of Objects	Contains fully rendered 3D product shots. Note: This array will only be populated if the product is 3D-enabled AND the internal 3D generation service has run successfully. Intended Use: Product Detail Pages (PDP).
Implementation Best Practices
To ensure the best user experience on your storefront, please follow these UI implementation guidelines:

1. Listing Pages (Catalogs & Grids)

Always use productThumb as the primary image for listing pages. Do not use this field for full-page hero images, as the resolution is optimized for thumbnails.
2. Product Detail Pages (PDP) - Fallback Logic

When rendering a Product Detail Page, you should implement the following fallback logic:
Primary Attempt: Check if the product3dImages array contains data. If so, load these as the primary hero images.
Fallback: If product3dImages is empty or null, fall back to loading the 2D templates from the productImages array.
WebHook
Overview
Webhooks are an API feature that allows your system to receive real-time notifications about certain events - without needing to poll the API repeatedly.

When an event occurs related to your store or data, the Helix server will make a POST request to your defined URL that will contain a JSON object in the request body. Your server has to respond with HTTP status 2xx OK, otherwise, the request will be retried in increasing intervals , max retires 3.

 Helix Platform ──── Event occurs ────► POST your-endpoint.com/webhook
                                        (JSON payload with event data)
The JSON object will always contain these attributes:

Field Name	Type	Description
EntityId	Integer	The unique ID of the object associated with the event.
EntityType	String	The category of the entity (e.g., order, product). See Entity Types for a full list.
EntityAction	String	The specific action that triggered the event (e.g., created, updated). See Entity Actions for a full list.
EventDateTime	TimeStamp	The timestamp indicating when the event was generated.
Description	String	A brief, human-readable summary of the event.
AdditionalData	Object	Context-specific metadata related to the event. The contents of this object depend on the EntityType and EntityAction combination.
RetryCount	Integer	The number of times the system has attempted to deliver this webhook event. Starts at 0 for the initial attempt.
StoreId	Integer	The unique ID of the store ,optional
StoreProductId	Integer	The unique ID of the store product id(DesignTemplate).
LatestStatus	String	Latest status of the order.
How to Register Your Webhook
To begin receiving event payloads, you must register your endpoint URL. Log in to your Contrado Store Account.

Contrado Store Account → API Integration → Webhook and Domain Registration section.
